Why activate a Python environment?
Solution:
    To avoid version conflicts.
    Imagine if Project A:
        needs NumPy 3.5
    Project B:
        needs NumPy 3.0
    Project C:
        needs NumPy 4.0

    If you do not activate a Python environment, it becomes difficult to manage multiple versions.
    So, if you activate it, you create a separate environment for each project so that you can install the required packages.
    This creates a separate environment for each project.
